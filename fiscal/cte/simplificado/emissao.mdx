## O quê / quando

Use o CT-e Simplificado para consolidar **várias prestações** (rotas em `det`) para **um único tomador**. Transmita via `CTeRecepcaoSimpV4` — não use `CTeRecepcaoSincV4` do CT-e padrão.

Em homologação, a razão social do tomador deve ser `CTE EMITIDO EM AMBIENTE DE HOMOLOGACAO - SEM VALOR FISCAL`.

Em produção, informe IBS/CBS com `->withIbsCbs(...)`. Em homologação, o SDK pode preencher o bloco de teste NT 2026.002 se você omitir o método.

Para tomador em UF SP em contingência SVC, prefira `indIEToma=9` (não contribuinte) sem tag `IE`. A combinação `indIEToma=2` + `IE=ISENTO` foi rejeitada pela SVC-SP (`617` / `482`) nesta campanha.

## Exemplo — emissão normal (tpCTe=5)

```php

use Suot\Fiscal\Cte\Data\Simplified\SimplifiedCteTaker;

try {
    $result = $cte->issueSimplified()
        ->forOperation('ot-consolidada-42-simp')
        ->inHomologation()
        ->issuedIn(new Uf('TO'))
        ->withIdentification(
            cfop: '6353',
            natOp: 'PRESTACAO DE SERVICO DE TRANSPORTE',
            serie: '1',
            number: '42',
            municipalityCode: '1721000',
            municipalityName: 'Palmas',
            originUf: 'TO',
            destinationUf: 'SP',
        )
        ->issuedBy(
            cnpj: '48014145000178',
            ie: '295269103',
            name: 'CTE EMITIDO EM AMBIENTE DE HOMOLOGACAO - SEM VALOR FISCAL',
            tradeName: 'EMITENTE HOMOLOG',
            street: 'Rua Teste',
            number: '100',
            district: 'Centro',
            zip: '77000000',
        )
        ->takenBy(new SimplifiedCteTaker(
            role: '4',
            ieIndicator: '2',
            cnpj: '48014145000178',

            name: 'CTE EMITIDO EM AMBIENTE DE HOMOLOGACAO - SEM VALOR FISCAL',
```

## Substituição (tpCTe=6)

Antes de `replacing()`, registre o evento do tomador Prestação em Desacordo (`610110`):

```php
use Suot\Fiscal\Core\Exception\InfrastructureException;

try {
    $prest = $cte->serviceInDisagreement()
        ->forOperation('ot-consolidada-42-prest-desacordo')
        ->inHomologation()
        ->accessKey($chaveCteSubstituido44)
        ->authorCnpj($cnpjTomador)
        ->observation('Prestacao do servico em desacordo para substituicao do CTe')
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('serviceInDisagreement');
    // Host: alertar; filar retry conforme $failure->retryPolicy
    throw $e;
}

if (! $prest->isRegistered()) {
    // Host: corrigir dados; sem 610110 a substituição tende a cStat 739
}

try {
    $result = $cte->issueSimplified()
        ->forOperation('ot-consolidada-42-sub')
        // …mesmos dados da emissão…
        ->replacing($chaveCteSubstituido44, $protocoloAutorizacao)
        ->handle();
} catch (InfrastructureException $e) {
    $failure = $e->toFailure('issueSimplified');
    // Host: alertar; filar retry conforme $failure->retryPolicy
    throw $e;
}

if ($result->isAuthorized()) {
    // Host: persistir XML, protocolo, chave do substituto; vincular ao CT-e original
} elseif ($result->isRejected()) {
    $error = $result->failure();
    // Host: corrigir dados ou cadastro; não reenviar cego
}
```

O protocolo do documento substituído fica no host (não vai no XML). O XML leva `infCteSub/chCte`.

<Tip>
Detalhe do evento: [Prestação em desacordo](/fiscal/cte/eventos/prestacao-desacordo).
</Tip>

<Warning>
SVC só autoriza `tpCTe=5` (NT 2024.002 G008). Não combine `replacing()` com `inContingency(SvcRs|SvcSp)`.
</Warning>

## Contingência

```php
use Suot\Fiscal\Cte\ValueObject\CteEmissionType;

$result = $cte->issueSimplifiedInContingency()
    ->forOperation('ot-consolidada-42-svc')
    // …dados da emissão (tpCTe=5)…
    ->inContingency(
        CteEmissionType::SvcRs,
        '2026-07-17T10:00:00-03:00',
        'Falha de comunicacao com a SEFAZ autorizadora',
    )
    ->handle();
```

## Como ler o Result

O retorno é o mesmo `IssueCteResult` do CT-e padrão (`isAuthorized` / `isRejected` / `isProcessing`, `failure()`, `retryPolicy()`). Persistência e política de retry ficam no host.
