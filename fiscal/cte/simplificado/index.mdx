O **CT-e Simplificado** (NT 2024.002) usa o mesmo modelo 57 do CT-e padrão, mas com leiaute `CTeSimp`, tipo `tpCTe` 5 (normal) ou 6 (substituição) e web service exclusivo `CTeRecepcaoSimpV4`.

Consolida várias prestações (rotas/`det`) para **um único tomador** — sem detalhar remetente/destinatário individuais no documento.

<Warning>
Homologação: emissão `cStat 100`, consulta, cancelamento e contingência SVC direta (`cStat 100`). Substituição (`tpCTe=6`) exige evento tomador Prestação em Desacordo (`610110`) antes de `replacing()` — esse evento ainda não está na API pública; a SEFAZ rejeita com `cStat 739` sem ele. Não declare production-ready absoluto.
</Warning>

## Quando usar

- Várias entregas/prestações para o **mesmo tomador**
- Carga fracionada com NF-e vinculadas por rota (`det/infNFe`)
- Não use o Simplificado quando precisar do leiaute completo com rem/dest/exped/receb — nesse caso use [`$cte->issue()`](/fiscal/cte/emissao)

## Operações

| Fluxo | Entry | SOAP |
|-------|--------|------|
| Status | `$cte->serviceStatus()…->handle()` | `CteStatusServicoV4` |
| Emissão Simplificado | `$cte->issueSimplified()…->handle()` | `CTeRecepcaoSimpV4` |
| Substituição | `$cte->issueSimplified()…->replacing($ch, $nProt)->handle()` | `CTeRecepcaoSimpV4` (tpCTe=6) |
| Contingência SVC/EPEC | `$cte->issueSimplifiedInContingency()…->inContingency(…)->handle()` | `CTeRecepcaoSimpV4` (tpEmis 4/7/8; SVC só tpCTe=5) |
| Consulta | `$cte->byAccessKey()…->handle()` | `CteConsultaV4` |
| Cancelamento | `$cte->cancel()…->handle()` | `CTeRecepcaoEventoV4` (`110111`) |
