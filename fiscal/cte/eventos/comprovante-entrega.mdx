## O quê / quando

CT-e autorizado com entrega concluída. Datas no fuso `America/Sao_Paulo` — `dhEntrega` futura ou em UTC costuma gerar rejeição (ex.: cStat 872).

## Fluent

```php
use Suot\Fiscal\Cte\ValueObject\CteProofOfDeliveryPayload;

$now = (new DateTimeImmutable('now', new DateTimeZone('America/Sao_Paulo')))
    ->format('Y-m-d\TH:i:sP');

$payload = new CteProofOfDeliveryPayload(
    authorizationProtocol: $nProt,
    deliveredAt: $now,
    receiverDocument: '12345678901',
    receiverName: 'Recebedor Exemplo',
    deliveryHash: base64_encode(hash('sha1', $chave44 . $now, true)),
    hashGeneratedAt: $now,
    deliveredNfeKeys: [], // chaves NF-e entregues, se houver
);

$result = $cte->proofOfDelivery()
    ->forOperation('pod-1')
    ->inHomologation()
    ->accessKey($chave44)
    ->emitterCnpj($cnpj)
    ->proofOfDelivery($payload)
    ->handle();
```

## Command equivalente

```php
use Suot\Fiscal\Cte\Application\Command\ProofOfDelivery\RegisterCteProofOfDeliveryCommand;

$command = RegisterCteProofOfDeliveryCommand::create()
    ->forOperation('pod-1')
    ->inHomologation()
    ->accessKey($chave44)
    ->emitterCnpj($cnpj)
    ->proofOfDelivery($payload)
    ->build();

$result = $cte->handle($command);
```

## Result

`ProofOfDeliveryCteResult` — `isRejected()`, `failure()`, `protocolNumber` do evento.
