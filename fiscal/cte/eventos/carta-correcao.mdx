## O quê / quando

Corrija campos permitidos de um CT-e autorizado sem cancelar. Cada item é um `CteEventCorrection` (`grupoAlterado`, `campoAlterado`, `valorAlterado`).

## Fluent

```php
use Suot\Fiscal\Cte\ValueObject\CteEventCorrection;

$result = $cte->correct()
    ->forOperation('cce-1')
    ->inHomologation()
    ->accessKey($chave44)
    ->emitterCnpj($cnpj)
    ->corrections([
        new CteEventCorrection(
            changedGroup: 'ide',
            changedField: 'xObs',
            changedValue: 'Observacao corrigida em homologacao',
        ),
    ])
    ->handle();
```

Construtor: `changedGroup`, `changedField`, `changedValue`, e opcionalmente `changedItemNumber` (coleções).

<Warning>
Só corrija grupos/campos permitidos pelo MOC/schema. Valores fiscais críticos (valores, chave vinculada, etc.) em geral exigem cancelamento e nova emissão — valide na especificação oficial.
</Warning>

## Command equivalente

```php
use Suot\Fiscal\Cte\Application\Command\Correct\CorrectCteCommand;

$command = CorrectCteCommand::create()
    ->forOperation('cce-1')
    ->inHomologation()
    ->accessKey($chave44)
    ->emitterCnpj($cnpj)
    ->corrections([/* CteEventCorrection… */])
    ->build();

$result = $cte->handle($command);
```

## Result

`CorrectCteResult` — `isRejected()`, `failure()`, protocolo do evento quando OK.

Exemplo executável (stub): `examples/cte/correct-event.php` no repositório do SDK.
