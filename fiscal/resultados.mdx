Toda operação pública de CT-e devolve um `*Result` tipado. Rejeição fiscal **não** é exception.

## Estados

| Situação | Como tratar |
|---|---|
| Autorizado / registrado | `isAuthorized()` (emissão) ou campos de sucesso do Result |
| Rejeição SEFAZ / fiscal | `isRejected()` → `failure()` → `retryPolicy()` |
| Processando / ambíguo | `isProcessing()` ou consultar (`QueryBeforeRetry`) |
| Infra (timeout, rede, cert) | Exception — `try/catch` separado |

## Exemplo (emissão)

```php
$result = $cte->issue()/* … */->handle();

if ($result->isAuthorized()) {
    // accessKey, protocolNumber, submittedDocumentXml, responseXml
}

if ($result->isRejected()) {
    $failure = $result->failure();
    $guide = $failure?->recoveryGuide;
    $policy = $result->retryPolicy();
    // $policy->mode: never | immediate | after_delay | after_correction | query_before_retry
}
```

O mesmo padrão (`failure()`, `retryPolicy()`, `canRetry()`) aparece em cancelamento, eventos, EPEC, contingência, consultas e `downloadProc`.

## O que o host faz

- Persistir artefatos do Result ([CT-e → Resultados](/fiscal/cte/resultados))
- Executar retry conforme [Retry](/fiscal/retry) — o SDK **não** reenvia sozinho
- Corrigir dados com `IssueCteCommand::revise()` quando `AfterCorrection`
