## O quê / quando

A SEFAZ (ou validação fiscal) recusou a operação. O Handler devolve Result rejeitado com código e mensagem. Use a orientação acionável antes de reenviar.

## Ler a rejeição

```php
if ($result->isRejected()) {
    $failure = $result->failure();
    // $failure->code, $failure->message
    // $failure->recoveryGuide->summary
    // steps com relatedField (caminho público, ex.: shipper.stateRegistration)

    $mode = $result->retryPolicy()?->mode;
}
```

## Corrigir e reemitir

```php
use Suot\Fiscal\Cte\Data\CteParty;

$retryCommand = $command
    ->revise()
    ->withOperationId($novoOperationId)
    ->forShipper($shipperCorrigido)
    ->build();

$result = $cte->handle($retryCommand);
```

<Warning>
Não reutilize o XML assinado anterior após alterar dados fiscais. Gere novo documento com novo `operationId`.
</Warning>

## Não confundir

| Rejeição fiscal | Infraestrutura |
|---|---|
| Result + `failure()` | Exception |
| Corrigir / consultar / aguardar | Retry de rede, cert, timeout com política do host |

Detalhes de modos: [Retry](/fiscal/retry). Exemplo: `bin/dx/06-rejection-and-revise.php`.
